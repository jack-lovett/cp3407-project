# User story title: View restaurant menu

## Priority: 10
High-level (iteration-agnostic).

## Estimation:
TBD (planning poker).

## Assumptions (if any):
- Menu items include at minimum: name, price, description (optional image).
- Filtering can start simple (e.g., keyword search, dietary tags, price range).

## Description:
Description-v1: As a **customer**, I want to view a restaurant's menu (items + prices + descriptions) and filter that menu to my particular queries, so I can find what I want quickly.

## Tasks
1. Define menu item fields.
2. Define filter options for v1.
3. Create menu view UI per restaurant.
