# Persona: Restaurant / Service Provider

- **Role / Type:** Restaurant owner/manager
- **Primary goal:** Receive and fulfil orders smoothly with minimal disruption.
- **Secondary goals:** Keep menu/profile accurate; maximise sales; manage busy periods.
- **Context:** On a tablet/desktop in-store during service.
- **Behaviours:** Monitors incoming orders; updates prep status; edits menu items.
- **Pain points:** Sudden order spikes; menu changes are painful; unclear customer issues.
- **Motivations:** More orders, better efficiency, fewer disputes.
- **Tech comfort:** medium

## Core jobs-to-be-done
- When orders come in during a rush, I want a clear queue and simple status updates, so I can keep fulfilment accurate and fast.

## Key user journeys (high-level)
- Set availability/hours → manage menu/profile.
- Receive order → accept/prepare → mark ready/complete → handle issues.

## Top needs / priorities
- P10: Local discovery listing + profile control.
- P20: Update menu easily.
- P40: Track incoming orders clearly + mark prepared/complete.
