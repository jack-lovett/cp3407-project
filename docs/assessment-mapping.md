# Assessment mapping (rubric → evidence)

This page maps the marking rubric criteria to where we provide evidence.

## Rubric criteria
1. **Requirements** → `requirements.md` + `../User_stories.md` + `../user_stories/`
2. **Design** → `design.md` (architecture/DB/UI links + screenshots)
3. **Implementation / Code** → `implementation.md` + repo source (`../src/`)
4. **Test** → `testing.md` + test code (`../tests/`)
5. **Version Control** → `version-control.md` (branches/PRs/issues/commit practices)
6. **Building & Dev Tools** → `tools.md` (libraries, build tooling, cloud services)
7. **Agile Software Engineering** → `agile.md` + iteration boards (`../iteration_1.md`, `../iteration_2.md`)
8. **Technical Writing** → `writing.md` (template + structure + consistency)

## Project artefacts
- Meeting minutes: `meetings/`
- Decision log: `../decisions.md`
- Task sheet: `task-sheet.md`
- Marking guide: `marking-guide.md`
